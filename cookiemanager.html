What is auto session remove?

Cookies on the browser will be removed immediately when the browser is closed.

The extension converts all currently existing cookies and also cookies added thereafter to session cookies. By its default definition, session cookies only last for the session and expire after you close the browser.

What is manual remove and when should I use it?

Manual remove lets you select cookies from specific domains and delete them.

It is useful when you find that you only want to remove some or all the cookies from a certain domain such as your Gmail or Facebook but want to retain the rest of the cookies.

What does the undo last remove button do?

If you find that you've accidentally removed cookies from the computer and want to reverse the changes, this will let you do just that.

Why and how should I add cookies to Chrome?

Cookies are pieces of information stored locally on your computer by the websites that you visit to keep track of who you are and other information about you.

You can add cookies with this extension if you know the values that you want to include. For example, here are the different values that a cookie has. Values marked with * are mandatory when you are adding new cookies.

Domain*: This is the domain of the cookie. For example, if you are adding a cookie for Facebook, you can set this to www.facebook.com

Name*: Cookies have a name value that identify it.

Value: These are the contents of the cookie and the information the server keeps on you.

Path: Some cookies are path specific to a domain, which means that they are only used on some parts of the website. Not very common but is used by some sites.

Lifetime: You can set the expiration of the cookie in three different ways.
- Select the date from a calendar style popup.
- After many days it will expire from today.
- Session (which will expire as soon as the browser is closed).

Secure: Cookies set with the secure flag will only be transmitted when there is a HTTPS secure connection between the website and the user. In other words, the browser will prevent the transmission of a cookie over an unencrypted channel.

HttpOnly: This is also another security property in cookies set by the website which helps mitigate the risk of client side script accessing the protected cookie.

When should I import/export cookies?

An example would be a time when you want to let a friend sign in to a website that you use without giving that person your username and password.

From the website that you would like to extract the cookie information from, choose the export tab. Then choose either copy to clipboard or export to file.

For exporting to file, you can also set a password which will be then used to encrypt the cookie information. This is useful if you are sending the file over an unencrypted channel where unauthorized parties may be able to read the cookie file.

The steps are similar when importing the cookie data back in. You can paste it from your clipboard, or import the exported file. If it was password encrypted during the export, you must enter the same password when importing it back in.
